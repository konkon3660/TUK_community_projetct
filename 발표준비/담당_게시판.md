# 게시판 담당 커닝페이퍼 (세로로 다 걸침: 화면 → 서버 → 저장)

> 내 파트는 화면 하나가 아니라 **게시판 기능 전체 흐름**이다. 아래 파일들이 다 내 것.

## 내가 건드린 파일
- **화면(client/GUI):** `BoardMenuPanel`(대분류 선택), `PostListPanel`(목록+검색),
  `PostDetailPanel`(상세·이미지·댓글), `PostEditorPanel`/`GroupBuyPostEditorPanel`/`NoticePostEditorPanel`(작성)
- **모델(model/boards):** `Post`(부모) + `NoticePost`/`GroupBuyPost`/`ComplaintPost`(상속),
  `AbstractBoard` + `FreeBoard`/`DepartmentBoard`/`DormBoard`/`NoticeBoard`/`GroupBuyBoard`/`ComplaintBoard`, `Comment`
- **서버(server/CT):** `ClientHandler`의 `handlePostList/Create/Update/Delete`, `handleCommentAdd/Delete`, `handleFileUpload/Download`
- **프로토콜:** `POST_LIST/CREATE/UPDATE/DELETE`, `COMMENT_ADD/DELETE`, `FILE_UPLOAD/DOWNLOAD`

## 내 기능 한 줄 흐름
글쓰기 버튼(화면) → `Packet.request(POST_CREATE, ...)` → 서버 `handlePostCreate`가 검증·저장 →
`board.save()`로 `.dat` 파일 기록 → 성공 응답 → 목록 화면 갱신.

## 무조건 방어할 질문 6개 + 답

**Q1. 게시판이 자유/학과/기숙사/공지/공동구매/민원 6종인데 코드를 어떻게 구성했나요?**
> 게시글은 `Post`를 부모로 두고, 추가 필드가 있는 것만 상속받습니다 — 공지는 대상 학과,
> 공동구매는 최대 인원·채팅방 id, 민원은 답변 상태. 게시판도 `AbstractBoard`가 목록·저장·권한 위임
> 같은 공통 동작을 다 구현하고, 자식 게시판은 "누가 볼 수 있나(canAccess)"와 "한 줄을 글로 복원
> (parsePost)"만 채웁니다. **상속·다형성**으로 종류가 늘어도 중복이 안 생깁니다.

**Q2. (핵심) 관리자가 남의 글을 수정할 수 있나요?**
> 아니요. `Post.canEdit()`은 **본인 글일 때만 true**라서 관리자도 남의 글 수정은 못 합니다.
> 다만 부적절한 글을 **삭제**하는 건 운영상 필요해서 `canDelete()`는 관리자에게 허용합니다.
> "글은 기록이라, 고쳐 쓰기가 아니라 내리기"라는 원칙입니다. 그리고 이 판단을 **화면에서 버튼만
> 숨기는 게 아니라 모델(canEdit)에서 하고, 서버도 같은 canEdit을 호출**해서 검사합니다 —
> 버튼만 숨기면 서버가 뚫려 있으니까요.

**Q3. 공지는 어떻게 대상 학과인 사람에게만 보이나요?**
> 공지 글에 대상 학과·기숙사 정보가 있고, 목록 요청 때 서버 `handlePostList`가
> `NoticeBoard.getVisiblePosts(현재유저)`로 **그 유저에게 보여야 할 공지만 걸러서** 보냅니다.
> 대상을 비우면 전체 공지입니다. 새 공지가 등록되면 서버가 대상자들에게 **실시간 푸시**도 보냅니다.

**Q4. 큰 이미지를 올리면 화면이 넘치지 않나요?**
> 원본 비율을 유지한 채 표시 영역에 맞게 **축소**합니다(`getScaledInstance`, 축소 비율은 가로·세로 중
> 작은 쪽 기준). 작은 이미지는 확대하지 않고 원본대로 두고, 그래도 넘칠 수 있어서 `JScrollPane`
> 안에 넣습니다.

**Q5. 사용자가 제목에 `|`나 본문에 줄바꿈을 넣으면 저장 파일이 안 깨지나요?**
> 파일은 "한 줄 = 글 하나", 필드는 `|`로 나눕니다. 그래서 저장 전에 사용자 입력값에
> `DataFormat.encode()`를 씌워 특수문자·줄바꿈을 이스케이프하고, 읽을 때 `decode()`로 되돌립니다.
> 본문 줄바꿈이 특히 위험한데(한 글이 두 줄이 되면 다음 서버 시작 때 로드 실패) 그걸 `\n`으로 바꿔 처리합니다.

**Q6. (경계 질문) 공동구매 글을 쓰면 채팅방이 자동으로 생기던데?**
> 네. 공동구매 글은 참여자 채팅방이 반드시 필요해서, `handlePostCreate`가 글이 `GroupBuyPost`면
> **같은 요청 안에서** 채팅방까지 만들고 글에 그 방 id를 연결합니다. 글 저장과 방 생성을 하나의
> 요청으로 묶어서, 중간에 끊겨 "주인 없는 방"이 남는 걸 막았습니다. 여기서부터는 **채팅방 담당**과
> 이어지는 부분입니다.

## 이것만은 외우기
- `Post` 부모 + 상속 3종, `AbstractBoard`가 공통 동작 → **상속·다형성**
- 수정은 **본인만**(canEdit), 삭제만 관리자 가능(canDelete) — **모델에서 판단, 서버도 같은 메서드**
- 저장 시 `DataFormat.encode` (특수문자·줄바꿈), 공동구매 글은 채팅방 자동 연결
