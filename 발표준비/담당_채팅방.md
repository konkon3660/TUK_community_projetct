# 채팅방 담당 커닝페이퍼 (세로로 다 걸침: 화면 → 서버 → 푸시 → 저장)

> 이 파트의 하이라이트 = **실시간 채팅(서버 푸시)**. 시연에서 제일 눈에 띄는 부분이라 제일 많이 찔림.

## 내가 건드린 파일
- **화면(client/GUI):** `ChatRoomListPanel`(목록·검색·가입신청·탭분리), `ChatRoomCreatePanel`(방 생성·제한 설정),
  `ChatRoomPanel`(채팅·닉네임·실시간 수신)
- **모델(model):** `ChatRoom`(참여자·가입신청·닉네임맵·제한조건), `Chat`(한 줄 메시지)
- **서버(server/CT):** `ClientHandler`의 `handleChatRoomCreate/JoinRequest/JoinApprove/JoinReject/ChatSend/
  ChatRoomSetNickname/ChatRoomList/ChatRoomDelete`, 그리고 푸시를 보내는 `SessionRegistry`
- **프로토콜:** `CHATROOM_CREATE/JOIN_REQUEST/JOIN_APPROVE/JOIN_REJECT/LIST/SET_NICKNAME/DELETE`,
  `CHAT_SEND`, `CHAT_MESSAGE_PUSH`

## 내 기능 한 줄 흐름
채팅 입력(화면) → `Packet.request(CHAT_SEND, ...)` → 서버 `handleChatSend`가 저장 →
방 참여자들에게 **서버가 먼저 push** → 상대방 `ChatRoomPanel.onPush`가 화면에 새 말풍선 추가.

## 무조건 방어할 질문 6개 + 답

**Q1. (핵심) 다른 사람이 보낸 채팅이 어떻게 실시간으로 내 화면에 뜨나요?**
> 요청-응답만으로는 안 되고 **서버 푸시**를 씁니다. 내가 `CHAT_SEND`를 보내면 서버 `handleChatSend`가
> 저장한 뒤, `SessionRegistry`로 그 방 **참여자들의 접속을 찾아 서버가 먼저 push 패킷을 보냅니다**.
> 받는 쪽 `ChatRoomPanel`은 `PushListener`로 등록돼 있어서 `onPush`가 호출되고, 새 메시지를 화면에 붙입니다.

**Q2. 그 push는 아무 스레드에서나 화면을 건드려도 되나요?**
> 아니요. push는 네트워크 스레드에서 오는데 Swing은 EDT(이벤트 디스패치 스레드)에서만 만져야 합니다.
> 그래서 `onPush` 안에서 `SwingUtilities.invokeLater(...)`로 감싸서 화면을 갱신합니다.
> (`ChatRoomPanel`에 실제로 그렇게 돼 있습니다.)

**Q3. 참여자가 아닌 사람이 그 방에 채팅을 보내면요?**
> 서버 `handleChatSend`가 `room.getMemberIds().contains(나)`를 확인해서, 참여자가 아니면
> "참여 중이 아닌 채팅방입니다" 예외를 던져 거부합니다. **권한 검사는 서버에서** 합니다.

**Q4. 방에 가입하는 절차는 어떻게 되나요?**
> 학생이 `CHATROOM_JOIN_REQUEST`로 가입 신청을 보내면 방장에게 신청이 쌓이고, 방장이
> `CHATROOM_JOIN_APPROVE`/`REJECT`로 처리합니다. 승인·거절은 **방장만** 가능하도록 서버가
> `room.getOwnerId()`를 확인합니다. 학과·학번·기숙사 같은 가입 제한도 방에 걸 수 있습니다.

**Q5. 닉네임은 어떻게 동작하나요?**
> 닉네임은 **방마다 다릅니다** — `ChatRoom`이 `Map<userId, 닉네임>`을 들고 있습니다(전역 프로필 아님).
> 입장 시 정할 수 있고 방 안에서 언제든 `CHATROOM_SET_NICKNAME`으로 바꿉니다. 안 정하면 학번을
> 그대로 보여줍니다. 채팅 표시할 때 닉네임이 있으면 그걸 우선 씁니다.

**Q6. 방 id는 클라이언트가 정하나요?**
> 아니요. `handleChatRoomCreate`에서 클라가 보낸 id는 무시하고 **서버가 001, 002... 순으로 채번**합니다.
> 클라가 정하게 두면 겹치거나 남의 방을 가리킬 수 있어서요. 공동구매 글에 딸린 방도 서버가 채번해
> 글에 연결합니다(여기서 **게시판 담당**과 이어집니다).

## 시연 하이라이트 (내가 보여줄 것)
클라이언트 2개 띄우고 → 각각 다른 계정 로그인 → 같은 방 입장 → **한쪽에서 친 채팅이 다른 쪽에
즉시 뜨는 것** + 닉네임 바꾸면 반영되는 것. 이게 이 프로젝트 최고 볼거리.

## 이것만은 외우기
- 실시간 = **CHAT_SEND → 서버가 참여자에게 push → onPush로 화면 갱신**
- push 화면 반영은 **`SwingUtilities.invokeLater`** 필수
- 참여자 검사·방장 검사·방 id 채번은 전부 **서버**가, 닉네임은 **방마다 Map**
