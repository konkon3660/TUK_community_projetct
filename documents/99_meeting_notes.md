## 07/22 — 팀플 세부 기능 사항 정리

> 기록: 김준서, 정세윤, 김현섭

### 회원가입/로그인 기능 세부

회원가입 입력정보
```
id : 2026591007
학과 : AI소프트웨어학과
기숙사생 여부 : O
비밀번호 설정 : pass01
is_admin: X
```

- 이미 있는 학번으로는 회원가입 못함.
- 아이디 누가 회원가입하면 훔쳐가면 관리자한테 문의 가능
- 제작, 전과, 기숙사 퇴실/입실 그런 정보변경은 관리자에게 문의.
- 관리자가 회원 정보 수정 가능.

### 향후 개선사항

- 복수전공 고려하기
- 학교도서관 책 추천 기능을 실제 한국공학대학교 중앙도서관 사이트에서 크롤링하여 가져오기
- 관리자 파트 분할 하여 문의 사항을 담당 부서가 관리하도록 개선 하기, 지금은 1개의 계정이라고 가정
- 각자 프로그램을 설치해야 하는 것이 아닌, 모바일과 웹으로 만들어서 접근성 향상.
- 할것 추천 기를 ai를 사용하여 세분화(여러명일 경우, 혹은 여러명인데 공강시간이 서로 다를 때)와 자동화를 시킨다.
- 룰베이스 알고리즘 기반 죽은 채팅방 채팅방 자동 관리
- 채팅에서 이미지나 파일 공유를 할 수 있도록 하자.

### 게시판 기능

게시판 객체
```
게시글 제목 : 작성자 설정
글쓴이 : id
게시글 내용 : 작성자 설정
파일 : 파일 경로
이미지 : 이미지 경로
댓글 : 댓글 객체들
```

댓글 객체
```
글쓴이 : id
댓글 내용 : 작성자 작성
```

**공통 기능**
- 관리자가 모든 글에 대해서 CRUD 가능.
- 사용자는 자신에 글에 대해서 CRUD 가능.
- 댓글 가능.
- 이미지 삽입 가능
- 파일 삽입 가능(크기는 5Mb)
- 게시판 글 검색 기능

**자유게시판**
- 모두가 이용 가능

**공동구매게시판**

공동구매 객체
```
게시판 객체를 상속받음.
최대 인원 수 설정 : 작성자 작성
채팅방 객체 : 채팅방 객체의 경로
해시태그들 : #기숙사|#음식
```

- 공동구매글을 작성할 때, 최대인원수 설정 가능.
- 자동으로 채팅방이 만들어지고 연결된다.
- 현재 인원수 = 채팅방의 인원수
- 해시태그 기능 추가. 해시태그는 정해져 있음.
- 해시태그 여러개 달 수 있음.
- 해시태그 별 필터링 가능. ex) 기숙사, 대량구매, 음식, 생필품 등

**학과별 게시판**
- 현존하는 학과 게시판
- User 속성에 있는 학과 정보를 통해서 권한 갈림.

학과별 게시판 객체
```
게시판에서 상속받음
접근제한 : 학과이름
```
- 관리자는 들어갈 수 있음.

**기숙사 게시판**
- 기숙사생만 이용 가능

기숙사 게시판 객체
```
게시판에서 상속받음
접근제한 : 기숙사 여부
```

**공지사항**
- 관리자가 쓴 글을 볼 수 있음.
- 관리자는 글을 쓸 수 있음.
- 관리자(시설관리팀, 학사운영팀)는 글을 쓸 때, 이 공지사항이 해당하는 학과/학부/기숙사 를 지정할 수 있음.
- 사용자는 필터링 가능.

공지사항 객체
```
게시판 상속
글 제목 :
글 내용 :
기숙사 공지 여부 :
해당하는 학과 : 구분자로 구분하여 여러개 저장 가능.
```

**민원 넣기**
- 사용자가 글을 관리자 한테만 보낼수 있고
- 학과 학번(로그인한 사용자 정보)이 같이 보내기게
- 관리자가 문의한 내용의 답을 사용자한테 보낼 수 있는 기능
- 자주 묻는 질문 탭에서 민원 템플릿을 제공함.
- 또한 문의 내역 탭에서는 문의한 내용을 볼 수 있고, 답변 대기(회색), 답변완료(파란색)으로 표기됨.
- 리스트를 클릭하면 문의내용과 그 밑에 문의 답변이 보여짐.

민원 객체
```
문의 카테고리 :
문의 2차 카테고리 :
문의 제목:
문의 이미지 첨부 : 이미지 경로
관리자 답변 여부 : True/false
```

- 템플릿 데이터 만들어두기
- 사용자가 GUI에서 템플릿을 불러와서 이어서 작성할 수 있다.
- 관리자가 볼 수 있는 민원 게시판.
  - 게시판을 상속받아서 구현.
  - 관리자만 볼 수 있고 접속 가능함.
  - 관리자가 답을 보낼 수 있도록.
  - 댓글 기능을 오버라이딩해서 관리자의 답변을 구현함.

### 채팅 기능(유저끼리)

- 오픈채팅방을 만들 수 있고 거기서 실시간 채팅을 할 수 있음.
- 채팅방을 만들 때 가입제한 O/X을 설정할 수 있음.
- 채팅방 제한: 입학년도(학번 앞 4자리), 학과제한

채팅방 객체
```
들어와있는 User의 id를 : ㄹㅇㄴㅁ
채팅 객체들 : 을 참조함.
방장의 user id : 2026591007
채팅방 가입 명 수 제한 : -1
입학년도 제한 :
학과(들) 제한 :
기숙사 여부 제한
초대수락 여부(O 라면 입학년도, 학과(들) 제한을 무시하고 초대를 넣을 수 있음)  :
```

채팅 객체
```
채팅을 보낸 시간 : 2026-07-23-13:31:31
채팅 내용 : 정세윤 바보
채팅 보낸 사람 : 2026591001
```

- 채팅방 참여를 할 때에는 가입지원 메세지를 볼 수 있음.
- 채팅방을 만든 사람이 수락/거절을 할 수 있음.
- 채팅방 입장시 들어가 면 프로필(이름만) 설정가능

ex) 채팅 예시 이미지
```
김현섭
  ㄴ>  나 바보다
```

- 최대 입장 가능 인원 설정

**채팅방 탐색**
- 채팅방 이름으로 검색
- 가입한 채팅방
  - 2-1. 공동구매게시판 탭
  - 2-2. 일반 채팅방

### 할 거 추천기

- 시간표 입력 기능을 따로 만들어서 시간표를 기입할 수 있도록. 언제든지 수정 가능
- 1교시는 9시 30분부터 1시간이고 2교시는 10시 30분 ….
- 시간표를 미리 입력해두면 오늘 날짜와 시간을 보고 오늘 공강 시간을 자동 인식하게끔.
- 할 거 추천 데이터에서 읽기만 함.
- 쓰거나 수정, 삭제가 없으므로, 구현하기 쉬울 것 같음. 그냥 파일 출력 끝.

할거 추천기
```
할 거 내용 : String : "노래방에 가기"
하는데 대강 걸리는 분 int: 30
```

- 할거 추천은 개발 할 때 생각해서 넣기로.
- 할거 추천은 범용성 있게 생각함
- 여러명이서 하든 혼자서 하든 포괄적으로 생각함.
- 또한 여러명이서 놀더라도 공강시간이 다를 수도 있지만 그것도 고려 하지 않는 걸로
- 할 것 범위는 49까지만
- 아래와 같은 식으로 30~40개의 할 일들을 나중에 개발할 때 생각해두기.

### 메뉴 추천기

- 오늘의 학식+ e동 정보 이미지 출력 -> 왜냐면 오늘 학식메뉴 뭐더라? 맛없네 뭐먹지 이런 생각.
- 맛 없을 경우: 오늘 맘스터치, 신북경, 북경마라탕, 토마토김밥, 한스오믈렛, 산융, 수호 식당,
  토스피아, 라온식당의 음식을 식당도 램덤 메뉴도 2개를 램덤
- 메뉴 추천 다시 받기 버튼을 만들기

- 책 이름이랑 관련된 과 내용이 저장된다

## 과제 배경 (원문)

```
Java 프로그래밍 응용 수업시간에 진행하는 조별 과제 시간.
각 조는 교수에게 프로젝트 주제를 심의하여 주제를 정하고, 프로젝트를 제작한다.
조원은 4명을 구성되어 있으며, 프로젝트 주제는 학교(한국공학대학교) 커뮤티니 프로그램으로 결정하였다.

생성형 AI를 활용하여 프로젝트를 개발하도 좋으나, 최대한 이해하고 배우는 방향으로 사용해야 한다.
또한 AI에게 사용한 프롬포트를 통합 제출해야 함.
따라서 프로젝트의 설계를 함께 진행한 뒤, 설계한 내용을 기반으로 각자 바이브 코딩을 하기로 결정하였다.
```
