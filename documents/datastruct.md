# 📌 팀 프로젝트 시스템 세부 기능 및 데이터 설계서

---

## 1. 파일 입출력(File I/O) 기반 데이터 관리 구조

DB 대신 로컬 파일 시스템을 활용하여 데이터를 관리합니다.

* **저장 방식:** 프로그램 실행 시 각 데이터 파일에서 읽어와 메모리(리스트)에 로드하고, 데이터 변경(생성/수정/삭제) 발생 시 해당 파일에 다시 저장합니다.
* **관리 파일 목록:**
* `users.txt` 또는 `users.dat`: 회원 정보
* `boards.txt` 또는 `boards.dat`: 전체 게시글 (자유, 공동구매, 학과, 기숙사, 공지, 민원)
* `chatrooms.txt` 또는 `chatrooms.dat`: 채팅방 목록 및 채팅 내역
* `schedules.txt` 또는 `schedules.dat`: 유저별 시간표 정보
* `recommendations.txt`: 할 거 / 메뉴 / 도서 추천 기초 데이터

## 2. 주요 데이터 객체(Entity) 명세

### 👤 회원 (User)

* **속성 목록:**
* `학번(id)`: 문자열 (예: "2026591007")
* `학과(department)`: 문자열 (예: "AI소프트웨어학과")
* `기숙사생 여부(isDormitory)`: 참/거짓 (O/X)
* `비밀번호(password)`: 문자열
* `관리자 여부(isAdmin)`: 참/거짓 (O/X)


* **핵심 규칙:**
* 이미 등록된 학번으로는 신규 회원가입 불가 (중복 체크)
* 계정 도용, 전과, 기숙사 입/퇴실 등 정보 변경은 관리자만 수정 가능



---

### 📝 게시판 및 게시글 (Board)

#### ① 기본 게시글 (공통 속성)

* **게시글 번호**: 고유 식별자
* **제목**: 작성자 설정
* **작성자 ID**: 작성자 학번
* **내용**: 작성자 설정
* **첨부 파일 경로**: 파일 저장 위치 (최대 5MB 제한)
* **첨부 이미지 경로**: 이미지 저장 위치
* **댓글 목록**: 댓글 객체들의 리스트
* **공통 권한 규칙:**
* 관리자: 모든 게시글/댓글에 대해 생성·조회·수정·삭제(CRUD) 가능
* 일반 사용자: 본인이 작성한 글/댓글에 대해서만 CRUD 가능 (타인의 글은 수정/삭제 버튼 비활성화)



#### ② 댓글 (Comment)

* **작성자 ID**: 작성자 학번
* **댓글 내용**: 작성자 입력
* **작성 시간**: 날짜 및 시간 데이터

#### ③ 세부 게시판 유형

* **자유 게시판**: 모든 회원 접근/이용 가능
* **공동구매 게시판** (기본 게시글 상속)
* `최대 인원 수`: 작성자 설정
* `연결된 채팅방 ID`: 게시글 작성 시 자동으로 전용 채팅방 생성 및 연결
* `해시태그 목록`: 미리 정해진 태그 선택 (예: #기숙사, #대량구매, #음식, #생필품)
* `특이사항`: 현재 인원수는 연결된 채팅방의 현재 인원수와 자동 연동, 태그별 필터링 기능 제공


* **학과별 게시판** (기본 게시글 상속)
* `접근 제한 학과`: 로그인한 유저의 학과 정보와 일치할 때만 접근 가능 (관리자는 전체 접근 가능)


* **기숙사 게시판** (기본 게시글 상속)
* `접근 제한`: 기숙사생 여부가 참(O)인 유저만 접근 가능 (관리자는 전체 접근 가능)


* **공지사항 게시판** (기본 게시글 상속)
* `작성 권한`: 관리자 전용
* `타겟 지정`: 특정 학과/학부 또는 기숙사 전용 공지 여부 지정 가능
* `사용자 기능`: 지정된 타겟 조건에 맞춰 필터링 조회



---

### 📬 민원 기능 (Complaint)

* **특징:** 기본 게시글 구조를 상속받아 관리자 전용 게시판으로 동작
* **속성 목록:**
* `문의 1차 카테고리` / `2차 카테고리`
* `문의 제목` 및 `문의 내용`
* `작성자 정보`: 로그인한 사용자의 학번/학과가 자동 포함되어 관리자에게 전달
* `이미지 첨부`: 이미지 경로
* `답변 상태`: 답변 대기(false, 회색 표시) / 답변 완료(true, 파란색 표시)


* **주요 기능:**
* 자주 묻는 질문 탭에서 작성용 민원 템플릿 제공
* 관리자는 민원 글의 댓글 기능을 통해 답글 작성 (답변 작성 시 '답변 완료'로 상태 변경)
* 사용자는 '문의 내역' 탭에서 자신이 넣은 민원과 관리자 답변 확인 가능



---

### 💬 채팅 기능 (Chat & ChatRoom)

#### ① 채팅 메시지 (Chat)

* **보낸 사람 ID**: 학번
* **보낸 시간**: 연-월-일-시:분:초
* **채팅 내용**: 텍스트 메시지

#### ② 오픈 채팅방 (ChatRoom)

* **방장 ID**: 개설자 학번
* **참여 유저 ID 목록**: 현재 들어와 있는 유저들
* **채팅 내역 목록**: 채팅 메시지 객체들의 리스트
* **최대 입장 인원 제한**: 수치 설정 (-1일 경우 제한 없음)
* **가입 제한 조건 설정:**
* `입학년도 제한`: 학번 앞 4자리 기준 (예: 2026학번 전용)
* `학과 제한`: 허용할 학과 목록 지정
* `기숙사 여부 제한`: 기숙사생 전용 여부
* `초대 승인 예외`: 초대를 받은 경우 위 제한 조건을 무시하고 가입 가능


* **가입 절차:**
* 채팅방 가입 신청 시 작성한 '가입지원 메시지'를 방장이 확인 후 수락/거절
* 입장 시 해당 채팅방에서 사용할 프로필(이름) 설정 가능



---

### 💡 추천 기능 명세 (Read-Only 데이터)

텍스트 파일 등에서 읽어와 화면에 출력하는 단순 조회 기능입니다.

1. **시간표 기반 할 거 추천기**
* 유저가 시간표(1교시 09:30~10:30 등)를 미리 입력해두면, 현재 날짜와 시간을 비교하여 오늘의 '공강 시간'을 자동 인식
* `추천 데이터 구성`: 할 일 내용(예: "노래방 가기"), 예상 소요 시간(예: 30분)
* 공강 시간에 맞춰 적절한 범위(약 30~40개 항목 내)의 할 거리를 단순 선별하여 추천


2. **메뉴 추천기**
* 오늘의 학식 및 e동 정보 이미지 출력
* 학식이 마음에 안 들 때를 대비해 주변 식당(맘스터치, 신북경, 북경마라탕, 토마토김밥, 한스오믈렛, 산융, 수호 식당, 토스피아, 라온식당 등) 중 식당 및 랜덤 메뉴 2개를 무작위 추천
* '메뉴 추천 다시 받기' 버튼 제공


3. **책 추천기**
* 유저의 학과에 맞춰 관련된 추천 도서 목록(책 이름, 연관 전공 내용) 중 무작위 1권 추천
